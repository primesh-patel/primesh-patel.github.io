AI Tools

The AI Tools Every Data Scientist Should Know in 2025

March 2025· 8 min read

The AI tooling landscape has changed dramatically. Knowing Python and scikit-learn alone isn't enough — the real productivity multiplier is knowing which AI tools to layer on top.

1. GitHub Copilot & Cursor

These handle boilerplate-heavy tasks effortlessly — data loaders, preprocessing pipelines, visualisation code. Treat them like a fast junior developer who needs clear instructions.

2. Claude & ChatGPT for Analysis

LLMs are now integral for explaining statistical concepts, reviewing code logic, and drafting project reports. Claude handles long notebooks without losing context — ideal for documentation.

3. AutoML: AutoGluon & FLAML

Tools like AutoGluon produce strong baseline models in minutes. Use them to benchmark what's achievable — then beat that baseline with domain knowledge and feature engineering.

4. Weights & Biases

W&B makes it easy to log hyperparameters, metrics, and artefacts across runs — saving hours of debugging.

5. SHAP for Explainability

A model nobody understands is a model nobody trusts. shap is now standard — especially for business-facing projects like insurance risk modelling.

6. Streamlit for Deployment

Stakeholders want to see models in action. Streamlit lets you build interactive dashboards in pure Python.

Conclusion

The best data scientists aren't those who avoid AI tools — they use them without sacrificing rigour. Start with one, master it, then add the next.

Real Estate Analytics: Web Scraping to Model Predictions

October 2024· 12 min read

A complete data science pipeline for Gurgaon's residential real estate market — from raw web scraping to a deployed Streamlit app with price prediction, analytics, and a property recommender.

The Problem

Real estate pricing in Gurgaon is opaque. Prices vary by sector, BHK, floor, furnishing, and dozens of other factors. I wanted a tool that predicted prices accurately and helped buyers understand the market.

Data Collection

Built a custom scraper using BeautifulSoup and Requests to collect 3,961 listings — 2,880 flats and 1,081 houses from 99acres.com.

11-Stage Data Pipeline

The pipeline covered outlier removal, missing value imputation, categorical encoding, area normalisation, luxury score engineering, and sector-level aggregation — resulting in 130+ engineered features.

Model Selection

Benchmarked Linear Regression, Ridge, Lasso, Random Forest, and XGBoost. Random Forest emerged as the best performer.

Key Learnings

  • Scraping quality directly determines model quality
  • Domain knowledge matters as much as ML technique
  • Feature engineering contributed more to performance than model selection

AXA Insurance: Predicting Claims with XGBoost & SHAP

December 2024· 10 min read

A machine learning system to predict insurance claim likelihood for AXA, identify high-risk segments, and produce interpretable outputs for underwriters.

The Business Problem

Flag high-risk customers before claims occur. Core challenge: severe class imbalance with only ~8% of policies resulting in claims.

Handling Imbalance with SMOTE

Used SMOTE combined with undersampling — dramatically improving recall without sacrificing precision.

Model Development

XGBoost consistently outperformed Logistic Regression and Random Forest on AUC-ROC and F1. Tuned via Optuna.

Explainability with SHAP

SHAP values revealed driving history, vehicle age, mileage, and geographic zone as top predictors. Force plots enabled individual policy explanations for underwriters.

Key Takeaways

  • Imbalanced classification requires deliberate strategy
  • Always evaluate precision, recall, and F1 together
  • SHAP is essential for trust in regulated industries
