Scraping Gurgaon Real Estate Listings

Two separate datasets were scraped — one for Flats and one for Houses — capturing features like price, area, bedrooms, bathrooms, balcony, floor number, society, nearby locations, and amenities.

Stages 02 & 03
Data Preprocessing — Level 1 & 2

Transforming messy raw data into a clean, structured, analysis-ready dataset through two levels of cleaning.

🧹
Level 1 · Per-Dataset Cleaning
Initial Data Cleaning & Standardisation
Cleaning each dataset independently before merging
01
Duplicate Removal
Flats: 0 duplicates found. Houses: 13 duplicate rows removed → reduced to 1,082 rows.
02
Missing Value Analysis
Flats: additionalRoom had 1,333 missing. Houses: society had 619 missing. Facing column: filled nulls with 'NA'.
03
Price Normalisation
Removed "Price on Request" entries. Standardised all prices to Crores (₹). Removed ₹/sq.ft. symbols from price_per_sqft.
04
Column Cleaning
Dropped link & property_id (non-informative). Renamed 'area' → price_per_sqft for clarity. Cleaned society names: removed star ratings (★) and inconsistent formatting.
05
Type Conversion
Converted "3 Bedrooms" → integer 3. Floor strings like "4th of 4 Floors" → integer 4. Ground=0, Basement=-1, Lower=0.
06
Area Calculation
Created 'area' column using formula: area = (price × 10,000,000) ÷ price_per_sqft. Applied to both datasets consistently.

Stages 06 & 07
Outlier Detection & Missing Value Imputation
🎯
Stage 06 · Outlier Detection
Identifying & Handling Extreme Values
IQR-based bounds, visual verification, and targeted removal

Price outliers were identified using the IQR method. Lower Bound: -1.75 Cr (effectively 0), Upper Bound: 5.45 Cr. A total of 425 outliers were detected in the price column alone. Whisker plots and ECDF plots were used to visually confirm outlier boundaries. After log transformation, extreme skewness was reduced significantly.

🔍 Method: IQR-based bounds + log transformation for price. Visual confirmation via box plots and ECDF before removal.

Multiple regression models were trained and evaluated for property price prediction. The target variable was log-transformed price (in crores). Models compared using R² score and RMSE via cross-validation.
